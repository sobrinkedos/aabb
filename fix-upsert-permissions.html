🔧 Fix UPSERT Permissions

Solução para o erro 409 Conflict no salvamento de permissões

❌ Problema: Erro 409 Conflict

📊 Evidência do Erro:

POST /rest/v1/permissoes_usuario 409 (Conflict) Erro ao salvar permissões: [detalhes do erro]

🔍 Possíveis Causas:

  1. RLS bloqueando DELETE: Não consegue remover permissões existentes
  2. Constraint de chave única: Tentativa de inserir registro duplicado
  3. Permissões já existem: Tabela já tem dados para o usuário
  4. Transação incompleta: DELETE não executou mas INSERT tentou

🎯 Causa Mais Provável:

O DELETE está sendo bloqueado pelo RLS, então as permissões existentes não são removidas. Quando o INSERT tenta adicionar novas permissões, gera conflito com as existentes.

🔍 Solução 1: Logs Detalhados (Implementado)

✅ Logs Adicionados:

🗑️ Removendo permissões existentes... ✅ Permissões existentes removidas 💾 Inserindo X novas permissões... 📋 Dados para inserir: [array de permissões] ✅ X permissões salvas com sucesso

🎯 Objetivo:

Identificar exatamente onde o processo está falhando - se é no DELETE ou no INSERT.

💡 Solução 2: Usar UPSERT (Recomendado)

🔧 Código UPSERT:

// Em vez de DELETE + INSERT, usar UPSERT const { error: upsertError } = await supabase .from('permissoes_usuario') .upsert(permissoesParaInserir, { onConflict: 'usuario_empresa_id,modulo' });

✅ Vantagens do UPSERT:

  • ✅ Atômico: Uma única operação
  • ✅ Sem conflitos: Atualiza se existe, insere se não existe
  • ✅ Menos RLS: Não precisa de permissão de DELETE
  • ✅ Mais seguro: Não há estado intermediário
