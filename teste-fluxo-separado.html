🧪 Teste Fluxo Separado - Credenciais vs Permissões

Documentação do novo fluxo que separa criação de credenciais da edição de permissões

🎯 Objetivo: Permitir editar permissões sem criar novas credenciais

📊 Comparação dos Fluxos

❌ Fluxo Antigo (Problemático)

  1. Criar funcionário básico
  2. Criar credenciais + permissões juntos
  3. ❌ Para alterar permissões = criar novas credenciais
Problema: Não conseguia editar apenas permissões

✅ Fluxo Novo (Corrigido)

  1. Criar funcionário básico
  2. Criar credenciais (se necessário)
  3. ✅ Editar permissões (independente)
Solução: Permissões e credenciais são independentes

🔧 Implementação

1. Componente PermissionsModal.tsx

Localização: src/components/EmployeeModal/PermissionsModal.tsx

Função: Interface visual para configurar permissões específicas

  • ✅ Lista todos os módulos do sistema
  • ✅ Permite configurar permissões granulares (visualizar, criar, editar, etc.)
  • ✅ Presets rápidos (Operador Caixa, Atendente Bar, etc.)
  • ✅ Salva apenas permissões (não mexe em credenciais)

2. Hook useEmployeePermissions.ts

Localização: src/hooks/useEmployeePermissions.ts

Função: Gerencia permissões no banco de dados

  • ✅ getEmployeePermissions() - Busca permissões atuais
  • ✅ saveEmployeePermissions() - Salva permissões específicas
  • ✅ clearEmployeePermissions() - Remove permissões específicas
  • ✅ applyPermissionPreset() - Aplica presets

3. Botão na Interface

Localização: Tabela de funcionários em src/pages/BarEmployees/index.tsx

Novo botão: ⚙️ "Gerenciar permissões"

  • ✅ Ícone: <Settings> (engrenagem roxa)
  • ✅ Abre modal de permissões
  • ✅ Independente do botão de credenciais

🎯 Como Usar o Novo Fluxo

Cenário 1: Funcionário Novo

  1. Criar funcionário: Botão "Novo Funcionário"
  2. Criar credenciais: Botão 🏷️ "Criar credenciais" (se precisar de acesso ao sistema)
  3. Configurar permissões: Botão ⚙️ "Gerenciar permissões"

Cenário 2: Funcionário Existente (Só Alterar Permissões)

  1. Localizar funcionário na tabela
  2. Clicar em ⚙️ "Gerenciar permissões"
  3. Configurar módulos que ele pode acessar
  4. Salvar - sem mexer nas credenciais existentes

Cenário 3: Usar Presets

  1. Abrir modal de permissões
  2. Clicar em preset (ex: "Operador de Caixa")
  3. Salvar - aplica permissões pré-definidas

✅ Presets rápidos: Configuração rápida para roles comuns
